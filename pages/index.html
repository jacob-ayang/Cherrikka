CHERRIKKA

Cherry Studio ↔ RikkaHub 备份互转

主题

快速入口

在线转换站： https://uwp.de5.net

仓库地址： jacob-ayang/Cherrikka

说明：本 GitHub Pages 页面是项目介绍页，完整转换请使用上面的在线转换站。

核心能力

  1. 双向转换：Cherry Studio ↔ RikkaHub。
  2. 多备份整合：一次上传多个 ZIP（可混合 Cherry + Rikka）。
  3. 冲突策略可选：latest / first / target / source。
  4. 输出带 sidecar：manifest + 原始 source 包，便于追溯回转。

上手说明（前端转换站）

  1. 上传一个或多个备份 ZIP。
  2. 源格式建议选 auto。
  3. 目标格式按需求选 Cherry 或 Rikka。
  4. 多输入时可选配置主次策略（默认 latest）。
  5. 点击 Convert，下载转换结果。

Fork 自部署（Vercel / Pages）

  1. 先 Fork 本项目到你自己的 GitHub 账号。
  2. 在 Vercel 里导入仓库并完成部署。
  3. 或者启用 GitHub Pages 发布本目录静态页。

Deploy with Vercel

CLI 示例（多输入整合）

go build -o cherrikka ./cmd/cherrikka
./cherrikka convert \
  --input cherry.zip \
  --input rikka.zip \
  --output merged-rikka.zip \
  --from auto \
  --to rikka \
  --config-precedence latest

GitHub Pages 启用提示

  1. 进入仓库 Settings → Pages。
  2. Build and deployment 的 Source 选择 GitHub Actions。
  3. 重新运行 Deploy Intro Page to GitHub Pages 工作流。

如果你看到 deploy-pages 的 Not Found / 404，基本都是这里没有启用 Actions Source。
